# 8.1.2.5: Claims Debug Page

**Status:** DONE
**Type:** Task
**Story:** [8.1.2: Entra ID OIDC Sign-In](./8.1.2-entra-id-oidc-auth.md)

## Description

Add a `/account/claims` page that displays all claims from the current user's `ClaimsPrincipal`. This is essential for debugging token contents, verifying that app roles, OIDs, and scopes are present, and confirming the identity passthrough chain.

## Implementation

- `Components/Pages/Account/Claims.razor` at route `/account/claims`
- Renders a `FluentDataGrid` (or table) with columns: Claim Type, Value
- Groups claims by category (identity, roles, token metadata)
- Shows "Not signed in" message when unauthenticated
- Accessible from the nav menu under Observability group
