# 8.1.3: Fluent UI Blazor component library setup

**Status:** DONE
**Type:** Task
**Feature:** [8.1: Blazor Project Foundation](./8.1-blazor-project-foundation.md)
**Tags:** `+web-app` `+ui`

---

Install and configure the Fluent UI Blazor component library to provide an enterprise-grade, accessible UI foundation that supports white-labelling via design tokens.

## Implementation Steps

- [x] Add NuGet package: `Microsoft.FluentUI.AspNetCore.Components`
- [x] Register Fluent UI services in `Program.cs`: `builder.Services.AddFluentUIComponents()`
- [x] Add Fluent UI CSS and JS references to `App.razor` or `_Host.cshtml` (depending on template version)
- [x] Replace the default Blazor template layout with a Fluent UI `FluentLayout` or `FluentNavMenu`-based shell:
  - Top bar with app name, login display, and theme toggle
  - Side navigation with placeholder menu items (Tools, Resources, Prompts, Audit)
  - Main content area
- [x] Verify that Fluent UI components (buttons, cards, data grids) render correctly
- [ ] Add a light/dark mode toggle using Fluent UI's `FluentDesignTheme` component

## Acceptance Criteria

- [x] Fluent UI components render without console errors or missing styles
- [x] The app shell has a professional layout with navigation, header, and content area
- [ ] Light/dark mode toggle works
- [x] The layout is responsive (works at desktop and tablet widths)
