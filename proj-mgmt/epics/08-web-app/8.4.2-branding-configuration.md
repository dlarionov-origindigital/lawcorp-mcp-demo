# 8.4.2: Branding configuration (logo, name, colors)

**Status:** BACKLOG
**Type:** Task
**Feature:** [8.4: White-Labelling & Theming](./8.4-white-labelling.md)
**Tags:** `+web-app` `+ui`

---

Make the app's branding elements (name, logo, tagline) configurable via `appsettings.json` so the web app can be re-branded for different demos or client presentations without code changes.

## Implementation Steps

- [ ] Define a `BrandingConfig` record: `AppName`, `Tagline`, `LogoPath`, `FaviconPath`, `FooterText`
- [ ] Bind from `appsettings.json` section `Branding`
- [ ] Update the app shell layout to read branding from the injected config:
  - Header: `BrandingConfig.AppName` and `BrandingConfig.LogoPath`
  - Browser tab: `BrandingConfig.AppName` in `<title>` and `BrandingConfig.FaviconPath`
  - Footer: `BrandingConfig.FooterText`
- [ ] Default configuration:
  ```json
  {
    "Branding": {
      "AppName": "Law-Corp",
      "Tagline": "Corporate law. Corporately.",
      "LogoPath": "/images/lawcorp-logo.svg",
      "FaviconPath": "/favicon.ico",
      "FooterText": "Law-Corp LLP â€” Reference Architecture"
    }
  }
  ```
- [ ] Provide a second example branding for the generic theme

## Acceptance Criteria

- [ ] Changing `Branding:AppName` in config changes the displayed app name everywhere
- [ ] Changing `Branding:LogoPath` swaps the logo without code changes
- [ ] The default Law-Corp branding is visually polished
- [ ] No hardcoded brand strings exist in Razor components
