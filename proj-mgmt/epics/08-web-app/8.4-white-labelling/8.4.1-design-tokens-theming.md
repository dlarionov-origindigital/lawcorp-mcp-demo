# 8.4.1: CSS design tokens and theme configuration

**Status:** BACKLOG
**Type:** Task
**Feature:** [8.4: White-Labelling & Theming](./8.4-white-labelling.md)
**Tags:** `+web-app` `+ui`

---

Establish a design token system using Fluent UI Blazor's theming API and CSS custom properties, enabling theme changes without modifying component code.

## Implementation Steps

- [ ] Define a `ThemeConfig` record in the web app project with properties: `PrimaryColor`, `AccentColor`, `BackgroundColor`, `FontFamily`, `LogoUrl`, `AppName`
- [ ] Bind `ThemeConfig` from `appsettings.json` section `Theme`
- [ ] Use Fluent UI Blazor's `FluentDesignTheme` component to apply the configured colors at the app root
- [ ] Create a CSS custom property layer (`site.css`) that maps `ThemeConfig` values to CSS variables used by custom components
- [ ] Create two example theme configurations:
  - **Law-Corp** (default): Navy blue primary, gold accent, serif headings, Law-Corp logo
  - **Generic**: Neutral gray primary, teal accent, sans-serif headings, placeholder logo
- [ ] Add a theme selector (dropdown or toggle) that switches between configured themes at runtime

## Acceptance Criteria

- [ ] Changing `Theme:PrimaryColor` in `appsettings.json` changes the app's primary color without code changes
- [ ] Both example themes are visually distinct and professional
- [ ] Custom components use CSS custom properties (not hardcoded colors)
- [ ] Theme switch at runtime updates all visible components immediately
