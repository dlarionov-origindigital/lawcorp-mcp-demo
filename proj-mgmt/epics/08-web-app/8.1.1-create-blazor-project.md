# 8.1.1: Create Blazor Web App project and solution integration

**Status:** DONE
**Type:** Task
**Feature:** [8.1: Blazor Project Foundation](./8.1-blazor-project-foundation.md)
**Tags:** `+web-app` `+foundation`

---

Create the `LawCorp.Mcp.Web` project using the Blazor Web App template (.NET 9, Interactive Server render mode) and integrate it into the existing solution.

## Implementation Steps

- [x] Run `dotnet new blazor -n LawCorp.Mcp.Web -o src/LawCorp.Mcp.Web --interactivity Server` (or equivalent .NET 9 template)
- [x] Add the project to `src/LawCorp.Mcp.sln`
- [x] Add project references:
  - `LawCorp.Mcp.Core` — shared domain types (`IUserContext`, `AttorneyRole`, etc.)
- [x] Add NuGet references:
  - `Microsoft.Identity.Web` — Entra ID OIDC
  - `Microsoft.Identity.Web.UI` — login/logout UI scaffolding
  - `ModelContextProtocol` — MCP client SDK (`HttpClientTransport`, `McpClientFactory`)
- [x] Configure `Program.cs` with minimal middleware: routing, static files, antiforgery, Blazor components
- [x] Verify the app builds, starts, and renders the default Blazor template at `https://localhost:5001`
- [ ] Add the project to the CI build (if a build pipeline exists)

## Acceptance Criteria

- [x] `dotnet build src/LawCorp.Mcp.sln` succeeds with zero errors/warnings from the new project
- [x] `dotnet run --project src/LawCorp.Mcp.Web` starts and serves the default Blazor page
- [x] `LawCorp.Mcp.Core` types are accessible from the web app project (e.g., `AttorneyRole` enum compiles)
- [x] Interactive Server render mode is confirmed (check `App.razor` for `@rendermode InteractiveServer`)

## Notes

The MCP server (`LawCorp.Mcp.Server`) and the web app (`LawCorp.Mcp.Web`) are separate processes. They communicate over HTTP. Do not add a project reference from the web app to the server project.
