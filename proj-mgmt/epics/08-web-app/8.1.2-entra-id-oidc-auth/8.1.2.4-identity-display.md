# 8.1.2.4: Identity Display

**Status:** DONE
**Type:** Task
**Story:** [8.1.2: Entra ID OIDC Sign-In](./8.1.2-entra-id-oidc-auth.md)

## Description

Display the authenticated user's identity in two locations: the header `LoginDisplay` component and the home page identity card.

## Implementation

### Header (`LoginDisplay.razor`)
- `FluentPersona` with name initials and 28px avatar
- Display name label
- Role badge extracted from the `roles` claim (falls back to `ClaimTypes.Role`)
- "Sign out" link

### Home Page (`Home.razor`)
- `<AuthorizeView>` conditionally renders:
  - **Authorized:** Identity card with `FluentPersona` (48px), display name, role, and `preferred_username` (email)
  - **NotAuthorized:** `FluentMessageBar` prompting the user to sign in

### Claims Mapping

| JWT Claim | UI Element |
|---|---|
| `name` | Display name, persona initials |
| `roles` | Role badge (Partner, Associate, etc.) |
| `preferred_username` | Email address below the name |
| `oid` | Not displayed in UI; used server-side for identity resolution |
