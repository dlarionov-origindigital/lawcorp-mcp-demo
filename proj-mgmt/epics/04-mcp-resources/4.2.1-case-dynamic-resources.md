# 4.2.1: Implement case-related dynamic resources

**Status:** BACKLOG
**Type:** Story
**Feature:** [4.2: Dynamic Resources](./4.2-dynamic-resources.md)
**Tags:** `+resources` `+case-mgmt`

---

As an AI assistant,
I want to resolve case-specific URIs for contextual data,
So that I can provide case-specific context in conversations without invoking a tool.

| URI Template | Description |
|---|---|
| `lawcorp://cases/{caseId}` | Case details including team, status, key dates |
| `lawcorp://cases/{caseId}/documents` | All documents for a case |
| `lawcorp://cases/{caseId}/timeline` | Case event timeline |
| `lawcorp://cases/{caseId}/billing` | Billing summary for a case |

- [ ] Register all four URI templates
- [ ] All enforce case-level authorization (user must have case access)

## Acceptance Criteria

- [ ] Invalid caseId returns a clear not-found error
- [ ] Unauthorized access returns appropriate error
- [ ] `billing` resource returns same aggregation as `billing_get_summary` tool
