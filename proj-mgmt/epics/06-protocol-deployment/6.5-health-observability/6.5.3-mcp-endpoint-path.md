# 6.5.3: Move MCP Endpoint to `/api/mcp`

**Status:** BACKLOG
**Type:** Story
**Feature:** [6.5 Health Checks & Observability](./6.5-health-observability.md)

## Goal

Change the MCP Streamable HTTP endpoint from root (`/`) to `/api/mcp` so custom endpoints (health, OpenAPI) don't conflict with MCP routing and the endpoint is clearly identifiable behind API Management.

## Acceptance Criteria

- [ ] `app.MapMcp()` changed to `app.MapMcp("/api/mcp")`
- [ ] `docs/local-dev.md` updated with new MCP endpoint URL
- [ ] Claude Desktop config examples updated
- [ ] VS Code / Cursor MCP config examples updated
- [ ] MCP Inspector instructions updated
- [ ] Web App MCP client config updated
- [ ] E2E tests updated if applicable

## Notes

- **Breaking change** — existing MCP client configurations will need to be updated
- Microsoft's App Service tutorial uses `/api/mcp` as the recommended path
- API Management expects MCP on a known prefix for proxy routing
