# 6.2.2: Implement cursor-based pagination for all search/list tools

**Status:** BACKLOG
**Type:** Task
**Feature:** [6.2: Cross-Cutting Protocol Features](./6.2-protocol-features.md)
**Tags:** `+protocol`

---

- [ ] Standardize pagination across all search tools
- [ ] Implement cursor-based pagination (not offset-based)
- [ ] Default page size: 20, max: 100
- [ ] Return shape: `results`, `nextCursor`, `hasMore`, `totalCount`
- [ ] Apply to: `cases_search`, `documents_search`, `clients_search`, `contacts_search`, `time_entries_search`, `invoices_search`, `research_search_memos`

## Acceptance Criteria

- [ ] All listed search tools accept an optional `cursor` parameter
- [ ] All listed search tools return `nextCursor` when more results exist
- [ ] `hasMore` is `false` on the last page
- [ ] Page size defaults to 20 and is capped at 100
- [ ] Cursor is opaque to the client (base64-encoded or similar)
