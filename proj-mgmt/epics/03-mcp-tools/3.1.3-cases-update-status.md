# 3.1.3: Implement `cases_update_status` tool

**Status:** DONE
**Type:** Story
**Feature:** [3.1: Case Management Tools](./3.1-case-management-tools.md)
**Tags:** `+case-mgmt`

---

As a lead attorney on a case,
I want to update a case's status,
So that case progress is tracked accurately.

- [x] Parameters: `caseId`, `newStatus`, `reason?`
- [x] Validate status transitions: Active → OnHold | Closed | Settled; OnHold → Active
- [x] Only lead attorney or partner can update status
- [x] Create CaseEvent of type StatusChange
- [x] Write to audit log
- [x] Set CloseDate when status transitions to Closed or Settled

## Acceptance Criteria

- [x] Invalid status transitions are rejected with a descriptive error
- [x] Associates who are not lead are rejected with an error
- [x] A CaseEvent is created for every status change
- [x] Audit log entry written with user identity and reason
