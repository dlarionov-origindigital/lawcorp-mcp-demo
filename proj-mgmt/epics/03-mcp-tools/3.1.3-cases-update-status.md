# 3.1.3: Implement `cases_update_status` tool

**Status:** BACKLOG
**Type:** Story
**Feature:** [3.1: Case Management Tools](./3.1-case-management-tools.md)
**Tags:** `+case-mgmt`

---

As a lead attorney on a case,
I want to update a case's status,
So that case progress is tracked accurately.

- [ ] Parameters: `caseId`, `newStatus`, `reason?`
- [ ] Validate status transitions: Active → OnHold, Active → Closed, Active → Settled
- [ ] Only Lead attorney or Partner can update status
- [ ] Create CaseEvent of type StatusChange
- [ ] Write to audit log

## Acceptance Criteria

- [ ] Invalid status transitions are rejected with a descriptive error
- [ ] Associates who are not Lead are rejected with 403
- [ ] A CaseEvent is created for every status change
- [ ] Audit log entry is written with user identity and reason
